# @uprise/ui design-sync notes

Storybook-shape sync of `@uprise/ui` (created this run — no prior storybook; Storybook 8 + react-vite + `@tailwindcss/vite` added under `packages/ui/.storybook`).

## Build / config decisions
- `[GENERAL]` No `dist` — converter bundles from source via `--entry packages/ui/src/index.ts`, `--node-modules packages/ui/node_modules` (has react + radix).
- `[GENERAL]` CSS is Tailwind v4 CSS-first; esbuild can't run Tailwind. Compiled the DS stylesheet with the Tailwind CLI to `packages/ui/.ds-styles.css` and set `cfg.cssEntry: ".ds-styles.css"`. **`cssEntry` resolves against the package root and is contained to it** — it MUST live inside `packages/ui` (a `.design-sync/…` path is rejected "resolves outside the package"). Regenerate: `.ds-sync/node_modules/.bin/tailwindcss -i .ds-sync/ds-css-entry.css -o packages/ui/.ds-styles.css --minify`.
- `[GENERAL]` `cfg.provider: {component: "ToastProvider"}` — the `.storybook/preview` decorator imports `preview.css` (`@import "tailwindcss"`) which esbuild can't resolve, so decorator bundling fails; setting the provider explicitly skips it and wraps previews in ToastProvider (a bundle export).
- Excluded via `titleMap: null`: **TurnstileWidget** (renders null without a Cloudflare key + network) and **ToastProvider** (a provider — its stories fire toasts on click, so the static card is empty; still bundled as the provider wrap).
- `overrides`: `cardMode: "single"` for overlays (Dropdown, ConfirmDialog, FormDialog, FieldOnboarding); `cardMode: "column"` for wide/gallery stories (Alert, Card, Spinner).

## Rendering / grading learnings (solo phase — Button, Dropdown, FormInput, StatusBadge)
- `[GENERAL]` The pipeline is faithful: previews render the real components with the real tokens (brand blue `#465fff`, success/error/warning/surface containers), lucide icons, and form states — all matched Storybook.
- `[GENERAL]` **Preview cards render at a smaller scale than the storybook canvas** — this is a framing difference; judge the component, not its size.
- `[GENERAL]` **Overlays (Dropdown/dialogs): the preview correctly renders the OPEN state** (via the story's toggle-on-mount / `open=true`), while the storybook reference often shows only the closed trigger (the open state didn't persist in the static SB build). A preview rendering MORE than the gated reference is faithful → grade `match` with a note; do NOT downgrade to `close`.

## Screenshot-grade campaign (all 38 shipping components)
- Roster: 40 PascalCase exports − TurnstileWidget (renders null w/o a Cloudflare key) − ToastProvider (the provider wrap, `titleMap:null`) = **38 storied components shipped**.
- Result: **38/38 graded `match`** — 35 first-pass (solo Button/Dropdown/FormInput/StatusBadge + fan-out batches A–D), 3 dialog overlays via a true pair after the harness patch below. No component preview needed a hand-authored override (`.design-sync/previews/` is empty); the pipeline renders faithfully.
- `[GENERAL]` **Portal/fixed overlays weren't screenshot-gradable — fixed in the harness.** ConfirmDialog, FormDialog, FieldOnboarding render their (open) content into `document.body` (Radix `Dialog.Portal`) or a `position:fixed inset-0` layer, i.e. OUTSIDE `#storybook-root`. `compare.mjs`'s `captureStory` only screenshotted the root, so the SB side came back "no storybook root content" while the DS side (which already viewport-captures portals, line ~543) rendered the dialog → no pair. **Patched `captureStory`** (in `.ds-sync/storybook/compare.mjs`) to detect a visible portal/fixed overlay (body child outside the root, or a `position:fixed` layer with height) and, in that case, capture the VIEWPORT instead of the empty root — mirroring the DS side. Backward-compatible: normal components (content in root) and closed-portal triggers (Select, TooltipHint) still take the root screenshot. After the patch all 3 overlays produce true side-by-side pairs and grade `match`.
  - **RE-SYNC RISK:** this patch lives in the *staged* `.ds-sync/storybook/compare.mjs`. If a future skill run re-stages the scripts from the bundle, the patch is lost and these 3 overlays revert to "no root content" (blocked). Re-apply the same `overlay`-probe change to `captureStory`, or the fix should be upstreamed into the skill.
- `[GENERAL]` **Zero-width stories collapsed in the SB reference — fixed in the stories.** Skeleton `Block` (`h-32 w-full`) and StepProgress `FirstStep`/`Complete` (bar of `flex-1` segments in a `max-w-md`, no intrinsic-width content) rendered blank in Storybook's centred static layout because `w-full`/`max-w-md`/`flex-1` establish no *definite* width with no ambient width. Fixed by wrapping them in a fixed-width container (`w-80`, `w-[28rem]`) in `skeleton.stories.tsx` / `step-progress.stories.tsx`. Sibling stories were unaffected (they carry intrinsic width). The DS previews already rendered these fine (fixed-width cards), so only the reference was blank.

## Re-sync risks
- Fonts: the DS uses "Outfit" (loaded via next/font in apps); neither storybook nor the previews load it, so both fall back the same and the compare can't see it. Verify `[FONT_MISSING]` at the gate; add Outfit @font-face if end-users need the exact typeface.
- `packages/ui/.ds-styles.css` is gitignored + regenerated (command above) — a fresh clone must recompile it before a re-sync.
- `readmeHeader: .design-sync/conventions.md` — repo-authored design-system conventions (voice, tokens, composition patterns) stitched to the top of the generated README that the design agent reads first. Edit that file to change the guidance; it's committed with the config.
